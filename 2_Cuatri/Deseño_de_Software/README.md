# Deseño de Software

> Programación · Segundo curso — 2º cuatrimestre

---

## De que vai esta materia

Deseño de Software é a materia que responde á pregunta: *¿como se constrúe software de calidade a escala?* Non só que o código funcione, senón que sexa comprensible, manteñible e ampliable co paso do tempo e con varios desenvolvedores traballando nel.

É onde a programación se converte en enxeñaría de software.

---

## Contidos habituais

- Proceso de desenvolvemento: modelos en cascada, iterativos, áxiles (Scrum, Kanban)
- Requisitos: captura, análise, especificación, validación
- Modelado con UML: diagramas de clases, secuencia, casos de uso, estado, actividade
- Patróns de deseño: creacionais, estruturais e de comportamento (GoF)
- Arquitecturas de software: capas, MVC, microservizos, eventos
- Principios de deseño: SOLID, DRY, KISS, YAGNI
- Calidade do software: métricas, code smells, refactoring
- Testing: unitario, integración, TDD, cobertura de código
- Control de versións avanzado: ramas, integración continua (CI/CD)
- Documentación técnica

## Consellos

- Os patróns de deseño non son solucións que se aplican sempre — son solucións a problemas concretos. Aprende primeiro o problema, despois o patrón.
- UML é unha ferramenta de comunicación, non un fin en si mesmo. Un diagrama que non se entende non serve de nada.
- TDD (Test Driven Development) parece lento ao principio. Cando o dominas, é máis rápido e produce código moito máis limpo.
- O código que escribes hoxe terá que mantelo alguén (probablemente ti) dentro de seis meses. Escríbeo para esa persoa.
